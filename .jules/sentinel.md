## 2024-05-24 - [Avoid Unverified COM Release in Game Hooks]
**Vulnerability:** Attempted to fix a "COM leak" by adding a `Release()` call to an interface pointer obtained via a reverse-engineered `QueryInterface` implementation.
**Learning:** In many game engine hooks and internal tools, the memory management logic for interfaces (like those found via `GetPropertyManager`) does not strictly follow COM reference counting rules. Forcing a `Release()` can cause immediate crashes or Use-After-Free (UAF) vulnerabilities if the engine expects the pointer to remain valid for its own internal use or doesn't track references as standard COM objects would.
**Prevention:** Only use `Release()` on COM pointers if you are certain the specific API or implementation requires it (e.g., standard Windows/DirectX interfaces). For internal game objects or managers, favor observation and minimal interference with the object lifecycle unless a clear leak is proven and the interface's behavior is fully understood.
